# api-yandex-direct — Claude Code Skill

Скилл для работы с Яндекс.Директ API v5 через Claude Code: месячные отчёты, реальные поисковые запросы, минус-фразы, кампании.

## Что умеет

- Месячный отчёт по кампаниям: расход, клики, показы, CTR, CPA, конверсии
- **Поисковые запросы** — по чему реально показываются объявления
- Автокандидаты в минус-фразы (запросы без конверсий, траты >100₽)
- Список всех кампаний (активных + архивных) с бюджетами
- Проверка остатка баллов (Units)
- Async polling для больших отчётов
- Поддержка агентских аккаунтов (Client-Login + Use-Operator-Units)

## Установка

В Claude Code напиши:

```
установи скилл из https://github.com/salto-agancy/salto-api-skills
```

Claude скачает все три скилла (LiveSklad + Метрика + Директ) и разложит по `~/.claude/skills/`.

## Первый запуск

Просто скажи в чате:

> Директ: сколько потратили в марте

Claude сам всё настроит:

1. Если нет токена — поведёт через настройку OAuth-приложения Яндекса (один раз, 5 минут):
   - Откроет нужные страницы Яндекса в браузере
   - Подскажет что заполнить в форме создания приложения
   - Спросит у тебя в чате ClientID, Client Secret и 7-значный код подтверждения
   - Сохранит токен — он же будет работать для Метрики
2. Если токен есть, но не указан логин клиента — спросит «логин какого клиента смотрим» (логин кабинета Директа клиента, не агентский).
3. Запустит отчёт и покажет результат.

Терминал открывать не надо — Claude всё делает сам через свои инструменты.

## Дальше

Просто пиши обычным языком:

- «Сколько потратили в Директе за март»
- «Какие фразы сливают бюджет — собери минус-слова»
- «Какие реальные запросы приносят клики»
- «Почему CTR упал»
- «Список активных кампаний с бюджетами»
- «Проверь отчёт директолога: расход, конверсии, цена заявки»
- «Сколько баллов осталось в Директе»

## Особенности (что хорошо знать)

- В Директе **нет лимита запросов в секунду** — есть **дневной лимит баллов (Units)**. Каждый запрос = 1+ Units, ошибки = 20 Units. Скилл следит за остатком автоматически.
- Большие отчёты ставятся в очередь (статус 201/202). Скрипт ждёт автоматически и забирает результат когда готов.
- Для агентств: используется заголовок `Client-Login` для работы с конкретным клиентом и `Use-Operator-Units: true` чтобы баллы списывались с агентского аккаунта.

## Безопасность

- У каждого пользователя своё OAuth-приложение Яндекса. Скилл НЕ использует общее приложение автора.
- Токен хранится локально в `~/.config/yandex-direct/.env` (`chmod 600`)
- В git ничего не уходит
- Скилл открытый — можно проверить что данные идут только в Яндекс
