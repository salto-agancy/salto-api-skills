# Rate limits — система Units

Директ не использует привычный «N запросов в минуту». Вместо этого — **баллы (Units)**.

## Как это работает

Каждый метод API стоит разное количество Units:
- Get-запросы: обычно 1-5 Units
- Запросы с большим объёмом данных (keywords.get, bids.get): 10-50 Units
- Reports: 10+ Units в зависимости от размера

**Ошибки стоят 20 Units** (штраф).

## Где смотреть

В ответе любого метода есть header:
```
Units: <spent>/<remaining>/<daily_limit>
```

Пример: `Units: 150/3850/4000` — потрачено 150, осталось 3850 из 4000 дневных.

## Дневной лимит

Вычисляется **динамически** на основе активности аккаунта:
- Мало кампаний / трат → низкий лимит (от 4000)
- Много активных кампаний → выше (до миллионов)
- Агентство с Use-Operator-Units → баллы с агентского счёта

Нельзя повысить через поддержку — только активность.

## Параллельность

- Максимум **5 одновременных запросов** с одного аккаунта рекламодателя
- На 6-й — `UNITS_PER_MINUTE_LIMITED_ERROR`

## Что делать при исчерпании

- `1000: Too many requests per second` — подождать 1 сек
- `1001: Day limit of units exceeded` — ждать до полуночи МСК
- Следить за header `Units` и замедляться при <20% остатка

## Оптимизация

1. **Batching**: один запрос с 100 IDs вместо 100 запросов по 1
2. **Changes API**: вместо полной перевыгрузки — только изменения с timestamp
3. **Только нужные FieldNames**: каждое поле = 1+ Units для list методов
4. **Use-Operator-Units** (агентства): списывает с агентского, не с клиентского
5. **gzip**: Accept-Encoding экономит трафик, но не Units

## Units для Reports API

Reports особенный:
- Одна отчёт = 10 Units минимум
- Но **большие отчёты в очередь** (201/202) — не блокируют параллельные запросы
- Максимум **5 отчётов в очереди** одновременно
- Отчёты хранятся 5 часов, потом надо перезапрашивать

## Типичные коды ошибок

| Код | Что | Решение |
|-----|-----|---------|
| 8000 | Недостаточно units | Ждать лимит на завтра |
| 8400 | Превышен лимит параллельных | Подождать / замедлиться |
| 9000-9999 | Ошибки валидации | Проверить body |
| 500 | Временная ошибка API | Retry через 10-30 сек |

## Песочница (sandbox)

Для разработки без трат Units:
```
https://api-sandbox.direct.yandex.com/json/v5
```

Нужно создать песочницу в интерфейсе Директа. Отчёты там возвращают заглушки, но структура настоящая.
