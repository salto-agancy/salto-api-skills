# api-yandex-metrika — Claude Code Skill

Скилл для работы с Яндекс.Метрикой через Claude Code: визиты, цели, источники трафика, карточка Я.Бизнес, сравнение периодов, Logs API.

## Что умеет

- Месячный отчёт: визиты, пользователи, отказы, глубина, время на сайте
- Разбивка по источникам (реклама / поиск / прямые / соцсети)
- Только рекламный трафик + UTM-breakdown
- Топ городов (фильтр зарубежных ботов)
- Цели: список + достижения + конверсии
- Яндекс.Бизнес карточка: переходы в профиль, клики по телефону, маршруты
- Сравнение двух периодов одним запросом
- Logs API — выгрузка сырых визитов

## Установка

В Claude Code напиши:

```
установи скилл из https://github.com/salto-agancy/salto-api-skills
```

Claude скачает скиллы и разложит по `~/.claude/skills/`.

## Первый запуск

Просто скажи в чате:

> Метрика: покажи визиты сайта за март

Claude увидит что токена нет и сам поведёт через настройку:

1. **Откроет в браузере** страницу создания OAuth-приложения Яндекса
2. **Подскажет** что заполнить в форме (название, Redirect URI, какие права отметить)
3. **Спросит у тебя в чате** ClientID и Client Secret из карточки приложения
4. **Откроет** страницу авторизации
5. **Спросит** 7-значный код подтверждения, который покажет Яндекс
6. **Сохранит токен** в `~/.config/yandex-metrika/.env` и параллельно в `~/.config/yandex-direct/.env`
7. **Покажет список** твоих счётчиков и спросит какой считаем

Терминал открывать не надо. Все команды Claude выполнит сам, нужно только в браузере подтвердить и переслать в чат 2 пары значений (ClientID + Secret и 7-значный код).

Если уже есть OAuth-приложение Яндекса с правами `metrika:read` и `direct:api-light` — сразу присылай ClientID и Client Secret, шаг создания пропустится.

## Дальше

После настройки просто пиши обычным языком:

- «Метрика: визиты с рекламы за март»
- «Сколько звонков из карточки Яндекс.Бизнес за март»
- «Сравни март и февраль по сайту»
- «Покажи реальных посетителей из Москвы (без ботов)»
- «Какие цели чаще всего достигаются»

## Безопасность

- У каждого пользователя СВОЁ OAuth-приложение Яндекса. Скилл НЕ использует общее приложение автора.
- Креды хранятся локально в `~/.config/yandex-metrika/.env` (`chmod 600`)
- В git ничего не уходит (`.gitignore`)
- Скилл открытый — можно проверить что данные отправляются только в Яндекс
