# Создание OAuth-приложения Яндекса (5 минут)

Один раз создаёшь приложение, дальше используешь его для Метрики и Директа.

Делать это нужно один раз в жизни. Если у тебя уже есть приложение с правами `metrika:read` и `direct:api-light` — переходи сразу к скрипту `scripts/get_oauth_token.py`.

## Зачем

Скилл не использует общее приложение Salto Agency — у каждого пользователя свой `client_id` и `client_secret`. Это:
- Безопасно: твои токены и данные никто другой не видит
- Изолированно: никто другой не съест твой rate limit
- Под твоим контролем: можешь в любой момент отозвать доступ через кабинет

## Шаги

### 1. Открой страницу создания приложения

https://oauth.yandex.ru/client/new

Войди в тот аккаунт Яндекса, к которому привязаны твои счётчики Метрики и/или кабинет Директа.

### 2. Заполни форму

**Название сервиса** (любое):
```
Salto Reports
```
или другое осмысленное (например, `Мои отчёты Метрики`)

**Иконка**: можно загрузить любую квадратную картинку или пропустить.

**Платформы**:
- Выбрать **«Веб-сервисы»**
- В поле **Redirect URI** вписать ровно эту строку:
  ```
  https://oauth.yandex.ru/verification_code
  ```
  (это специальный URI Яндекса — он покажет 7-значный код прямо в браузере)

### 3. Выбери права (scopes)

Прокрути вниз до раздела **«Доступ»**. Найди и отметь:

**Для Метрики:**
- ✅ `Яндекс.Метрика` → `Получение статистики, чтение параметров своих и доверенных счётчиков` (это `metrika:read`)
- (опционально) `Управление параметрами своих и доверенных счётчиков` — если планируешь не только читать, но и менять настройки счётчиков

**Для Директа:**
- ✅ `Яндекс.Директ` → `Использование API Яндекс Директа (для получения данных и управления рекламой)` (это `direct:api`)
  
  ИЛИ
  
- ✅ `Использование API Директа в режиме чтения` (это `direct:api-light`, рекомендую если только отчёты)

Если не уверен — выбирай облегчённые варианты (`-light` / только чтение). Расширить права всегда можно позже, отредактировав приложение.

### 4. Создай приложение

Нажми **«Создать приложение»** внизу страницы.

### 5. Скопируй ClientID и Client Secret

После создания откроется карточка приложения. На ней будут:
- **ClientID** — 32 шестнадцатеричных символа (a-f, 0-9), пример формата: `xxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxx`
- **Client Secret** — тоже 32 символа

⚠️ **Client Secret не показывай никому** и не выкладывай в публичные репозитории. Это пароль твоего приложения.

### 6. Запусти скрипт получения токена

```bash
python3 ~/.claude/skills/api-yandex-metrika/scripts/get_oauth_token.py
```

Скрипт спросит ClientID и Client Secret, откроет страницу авторизации, попросит ввести 7-значный код подтверждения, и сохранит токен в `~/.config/yandex-metrika/.env` и `~/.config/yandex-direct/.env`.

## Если что-то пошло не так

### «Не вижу нужный scope для Директа»
В Яндексе scopes Директа называются по-разному в разных кабинетах. Ищи всё что связано со словом «Директ» и отмечай.

### «Не открывается страница верификации»
Проверь Redirect URI — он должен быть **точно** `https://oauth.yandex.ru/verification_code`, без пробелов, со схемой `https://`.

### «Уже есть приложение, но не помню секрет»
В кабинете https://oauth.yandex.ru/ найди своё приложение → нажми **«Получить новый Client Secret»**. Старый перестанет работать.

### «Хочу использовать с другого аккаунта Яндекса»
Создай отдельное приложение в нужном аккаунте. ClientID привязан к аккаунту-владельцу.

## Безопасность

- ClientID и Client Secret хранятся в `~/.config/yandex-metrika/.env` (`chmod 600`)
- В git не уходят (`.gitignore`)
- Скилл не отправляет твои креды никуда кроме самого Яндекса
- Скрипт открытый — можно проверить
